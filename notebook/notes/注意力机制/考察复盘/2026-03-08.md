# 注意力机制复盘（QKV 到梯度与复杂度）

## 背景
- 日期：2026-03-08
- 主题：QKV、缩放点积注意力、softmax、多头注意力、`O(n²)`、`sqrt(dk)` 与梯度直觉
- 学习目标：形成可复述的知识逻辑脉络，并把错题与追问沉淀成长期可复习材料
- 当前卡点：softmax 梯度直觉、`O(n²)` 细节、数值稳定性、题目完整作答能力

## 知识逻辑脉络
1. 起点问题：模型如何在长序列里决定“看哪里、拿什么信息”。
2. 核心概念链路：`Q` 提问、`K` 建索引、`V` 提内容，先匹配再聚合。
3. 数学机制链路：`QK^T -> 缩放 -> softmax -> 加权 V`，得到上下文表示。
4. 训练与梯度链路：logits 过大使 softmax 饱和，雅可比矩阵变小，反向梯度变弱。
5. 工程与复杂度链路：瓶颈来自 `n×n` 注意力矩阵，导致时间约 `O(n²·d)`、显存约 `O(n²)`。

## 题目全量对照（题目/你的答案/标准答案/错误点）
### 题目 1
- 题目：`Q/K/V` 各自语义是什么？为什么不能只用一个向量同时承担三者角色？
- 你的答案：`Q` 是问题，`K` 用来找相关内容，`V` 用来生成输出；三者目的不同。
- 标准答案：`Q` 决定“我要找什么”，`K` 决定“我能被怎样匹配”，`V` 决定“被选中后输出什么”；三者由不同线性投影学习不同子空间功能。
- 错误点：基本正确，但可补充“不同投影矩阵带来不同表示子空间”。
- 修正后一口气答案：三者不是语义重命名，而是由三套参数分别学习“查询、索引、内容”，因此不能简单合并成一个向量角色。

### 题目 2
- 题目：写出 `scaled dot-product attention` 完整公式并标注每步 shape。
- 你的答案：`score=softmax(QK^T/根号dk)*V`，`QK^T` 输出 `n*n`，最后 `n*dv`。
- 标准答案：`Attention(Q,K,V)=softmax(QK^T/sqrt(dk))V`；`Q(n,dk) K(n,dk) V(n,dv)`，`QK^T(n,n)`，`A(n,n)`，`AV(n,dv)`。
- 错误点：基本正确；建议写完整符号与矩阵乘法顺序。
- 修正后一口气答案：`Attention(Q,K,V)=softmax(QK^T/sqrt(dk))V`，形状流是 `(n,dk)*(dk,n)->(n,n)->(n,n)*(n,dv)->(n,dv)`。

### 题目 3
- 题目：为什么要除以 `sqrt(dk)`（从 softmax 饱和与梯度角度解释）？
- 你的答案：`QK^T` 方差随 `dk` 增大，softmax 两极化，出现梯度爆炸。
- 标准答案：除以 `sqrt(dk)` 是为控制 logits 尺度，避免 softmax 过饱和；饱和后梯度通常变小或接近 0。
- 错误点：把主要问题说成“梯度爆炸”不准确。
- 修正后一口气答案：`sqrt(dk)` 用于抑制 logits 方差膨胀，防止 softmax 早饱和，避免梯度信号衰减。

### 题目 4
- 题目：若不除 `sqrt(dk)` 且 `dk` 很大，logits 与训练会怎样？
- 你的答案：梯度爆炸。
- 标准答案：logits 更极端，softmax 更接近独热分布，许多位置梯度接近 0，训练变慢且不稳定。
- 错误点：方向错，主要是梯度变小而非爆炸。
- 修正后一口气答案：`dk` 变大而不缩放会让 softmax 饱和，导致有效梯度减少。

### 题目 5
- 题目：多头注意力带来什么能力？与“单头但更大维度”差异是什么？
- 你的答案：每个头关注不同点。
- 标准答案：多头通过多个独立投影在不同子空间并行建模不同关系（位置、语义、句法等），比单头更易学习互补模式。
- 错误点：方向对，但缺少“独立子空间与互补关系”。
- 修正后一口气答案：多头不是简单并行复制，而是多子空间互补建模，表达能力通常优于单头放大。

### 题目 6
- 题目：自注意力时间复杂度和显存复杂度分别是什么？`O(n²)` 卡在哪一步？
- 你的答案：对应 `N`。
- 标准答案：核心时间复杂度约 `O(n²·d)`，显存（注意力矩阵）约 `O(n²)`；瓶颈在 `QK^T` 与权重矩阵存取。
- 错误点：回答不完整。
- 修正后一口气答案：时间主要花在 `QK^T` 与 `A·V`，显存主要花在 `A(n,n)`，因此 `n` 主导二次增长。

### 题目 7
- 题目：长度 `n -> 2n`，最重计算量变几倍？
- 你的答案：4 倍。
- 标准答案：约 4 倍，因为二次项从 `n²` 变成 `(2n)²=4n²`。
- 错误点：无。
- 修正后一口气答案：最重部分按 `n²` 缩放，长度翻倍时约四倍。

### 题目 8

- 你的答案：不知道。
- 标准答案：每一行是“某个 query 对所有 key 的权重分布”，行和为 1。
- 错误点：未作答。
- 修正后一口气答案：行视角就是“一个 query 的注意力分配”。

### 题目 9
- 题目：为什么 softmax 前做 `x-max(x)`？会改变结果吗？
- 你的答案：不知道。
- 标准答案：防止 `exp` 上溢，提高数值稳定性；不会改变结果，因为 softmax 对整体平移不变。
- 错误点：未作答。
- 修正后一口气答案：`x-max(x)` 只平移不改相对差值，概率分布不变但计算更稳定。

### 题目 10
- 题目：长序列下 `O(n²)` 太贵，给出 3 类优化思路。

## 先纠错
- 关键纠错 1：`sqrt(dk)` 主要防“梯度变小”，不是主防“梯度爆炸”。
- 关键纠错 2：复杂度必须分时间与显存两部分回答。
- 关键纠错 3：`softmax(QK^T)` 行语义必须会说，这是注意力直觉核心。
- 关键纠错 4：`x-max(x)` 是面试高频数值稳定点，必须秒答。
- 关键纠错 5：优化思路要按“稀疏/线性/KV 压缩”分类表达。

## 追问深化
- 追问 1：为什么 softmax 导数里有 `i` 和 `j` 两种情况？
  - 你当时的疑问：为什么有 `∂p_i/∂z_i` 和 `∂p_i/∂z_j`，到底看哪个？
  - 深化解释：softmax 是向量到向量映射，导数是雅可比矩阵；对角项表示“自己变动对自己输出”，非对角项表示“其他 logit 变动对该输出的影响”。
  - 关键结论：训练真正关注的是 `∂L/∂z`，它由所有 `∂p_i/∂z_j` 通过链式法则汇总。
- 追问 2：为什么未饱和和饱和时导数量级差这么大？
  - 你当时的疑问：`z=[1,0,-1]` 与 `z=[10,0,-10]` 的梯度为什么差别巨大？
  - 深化解释：`p_i(1-p_i)` 在 `p_i` 接近 0 或 1 时趋近 0；当 logits 极端时，softmax 接近独热分布，雅可比元素整体变小。
  - 关键结论：softmax 饱和阶段常见问题是梯度难传，不是软最大这一层导致的爆炸。
- 追问 3：链式法则“合起来”最终是什么？
  - 你当时的疑问：`∂L/∂z` 如何由所有 `∂p_i/∂z_j` 汇总？
  - 深化解释：通式是 `∂L/∂z_j = Σ_i (∂L/∂p_i)(∂p_i/∂z_j)`；交叉熵特例可化简为 `∂L/∂z = p-y`。
  - 关键结论：记住“通式 + 交叉熵特例”就能覆盖面试和实现推导。

## 核心总结
- 注意力计算是“匹配 + 归一化 + 聚合”的三段式。
- `sqrt(dk)` 的本质是尺度控制与梯度健康。
- `softmax` 的梯度本质是雅可比矩阵，不是单一导数。
- 多头是多子空间并行建模，不是单头维度放大。
- 长序列优化核心是减少有效连边、降低核算复杂度、压缩上下文长度。

## 面试可背诵版（3-5句）
1. 注意力本质是每个 query 对所有 key 做匹配，再用归一化权重聚合 value。  
2. 标准公式是 `Attention(Q,K,V)=softmax(QK^T/sqrt(dk))V`，`QK^T` 产生 `n×n` 权重前 logits。  
3. 除以 `sqrt(dk)` 是为了控制 logits 方差，避免 softmax 饱和导致梯度信号衰减。  
4. 自注意力时间复杂度约 `O(n²·d)`，显存瓶颈在注意力矩阵 `O(n²)`。  
5. 多头注意力通过多子空间并行学习互补关系，通常比单头更强。  

## 关键公式与形状
- 公式：
  - `Attention(Q,K,V)=softmax(QK^T/sqrt(dk))V`
  - `∂p_i/∂z_i = p_i(1-p_i)`，`∂p_i/∂z_j = -p_i p_j (i≠j)`
  - `∂L/∂z_j = Σ_i (∂L/∂p_i)(∂p_i/∂z_j)`
  - 交叉熵特例：`∂L/∂z = p-y`
- 形状流：
  - `Q(n,dk)`，`K(n,dk)`，`V(n,dv)`
  - `QK^T -> (n,n)`
  - `A=softmax(QK^T/sqrt(dk)) -> (n,n)`
  - `AV -> (n,dv)`
- 数值稳定要点：
  - 先做 `x-max(x)` 再 softmax
  - 监控注意力分布是否过早接近独热
