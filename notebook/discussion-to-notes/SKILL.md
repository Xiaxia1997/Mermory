# 双笔记沉淀

## 工作流

1. 先识别中文知识点名称（例如“注意力机制”“RAG 检索增强”）。
2. 自动判定 `note_type`：
   - `学习`：以讲解、推导、知识梳理为主，未出现完整考题与评分对照。
   - `考察复盘`：出现题目、用户作答、标准答案、纠错、评分、追问复盘等评测链路。
3. 按 `note_type` 选择模板并落盘。
4. 确保笔记保留“知识逻辑脉络”，不是散点摘抄。
5. 若为考察复盘，必须保留全题对照链路：题目、你的答案、标准答案、错误点。

## 输出要求

- 默认使用中文。
- 笔记内容必须为纯中文。
- 必须按“逻辑脉络 -> 关键结论”的顺序组织内容。
- 必须包含 `## 面试可背诵版（3-5句）`。
- 若为考察复盘，必须包含 `## 题目全量对照（题目/你的答案/标准答案/错误点）`。
- 若某题未作答，明确标记“未作答”。
- 若有追问，必须写 `## 追问复盘`。

## 文件规则

- 根目录：`notebook/notes/<中文知识点>/`
- 学习笔记：`notebook/notes/<中文知识点>/学习笔记.md`
- 考察复盘：`notebook/notes/<中文知识点>/考察复盘/YYYY-MM-DD.md`
- 同一天多次考察：`YYYY-MM-DD-01.md`、`YYYY-MM-DD-02.md`
- 若用户继续追问同一场考察，更新同一文件并追加 `## 追问复盘（HH:mm）`
